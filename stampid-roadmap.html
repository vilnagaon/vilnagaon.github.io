💡 Proposed Answers to Open Questions

Board-ready recommendations for the 8 unresolved items — prepared for discussion on 2026-04-10

Question 1

Crypto Platform Choice

Start on Scroll (L2), plan migration path to Europeum
  • Scroll is production-ready today — EVM-compatible, low gas costs
  • Europeum (EBSI) is strategically aligned but not production-ready yet
  • Multi-chain support (Scroll / Europeum) is already planned in P3
  • SDK should abstract the anchoring layer from day 1 so switching chains is a config change, not a rewrite
Decision: Mathieu & Leila (EBSI implications)
Question 2

GitLab Setup Timeline

1–2 days — use GitLab SaaS (EU data residency), don't block P1
  • GitLab SaaS setup is a few hours: create group, CI/CD templates, branch protection
  • Self-hosted on Hosteur is overkill at this stage (2–3d)
  • GitLab offers EU-only data residency on SaaS plan
  • Guillaume’s availability is a bottleneck — Stéphane could handle setup alone if needed
Decision: Guillaume / Stéphane
Question 3

Europeum Compatibility

Design chain-agnostic anchoring now, evaluate Europeum in H2 2026
  • Europeum is still emerging — source docs position it as future alignment, not immediate dependency
  • Architecture already describes proof layer as independent from anchoring infra
  • Define an AnchoringProvider interface, implement Scroll first, add Europeum when its API stabilizes
  • Keeps the European sovereignty narrative credible without blocking delivery
Decision: Mathieu (EBSI lead)

Question 5

API Pentest Scope & Timing

Two pentests — lightweight in P2, full external in P3
  • P2: Internal / freelance consultant (~2–3d). Focus on auth flows, API validation, key management boundaries
  • P3: Full external pentest (~5d + remediation). Complete surface: API, smart contracts, wallets, front-end. Specialized firm (web3 + API)
  • Budget estimate: 5–10k€ for the full pentest
  • Non-custodial architecture limits the attack surface — scope is manageable
Decision: Leila (budget) & Stéphane (scope)
